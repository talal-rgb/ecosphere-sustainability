🔒 Encryption Migration Test Suite

Demonstrates localStorage → encryptedStorage upgrade, corruption handling, and failure modes.

Step 1: Simulate Pre-Upgrade State (Plain localStorage)

Before the security upgrade, user data was stored as plain JSON strings.

Step 2: Upgrade to Encrypted Storage

Initialize encryptedStorage. Existing plain data is automatically migrated and encrypted.

Step 3: Verify Encrypted Data

Confirm data is now stored in encrypted format (AES-GCM ciphertext).

Step 4: Readback Test

Read encrypted data back — should decrypt transparently and return original values.

Failure Mode: Corrupted Storage

Simulate data tampering — encryptedStorage should detect corruption and return null.

Failure Mode: Wrong Password

Re-initialize with wrong password — decryption should fail gracefully.

Failure Mode: Cleared Storage

User clears browser storage — should handle gracefully.
